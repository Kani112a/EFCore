Create ASP.NET Core Web App (Model-View-Controller) 
appsettings.json   ---> Initialize Connectionstring
Add model in that set columns
Create DbContext --> In this set database
Register DbContext   ---->Program.cs   --->Call the connectionstring  Tells EF Core: (Use PostgreSQL database)
Create Migration    --->Add-Migration InitialCreate
WHAT HAPPENS?

EF Core:

Checks model classes
Creates migration files
Generates SQL

Update Database:  Update-Database

Next:
Create Controller

Right click:

Controllers
→ Add
→ Controller

Choose:

MVC Controller with views, using Entity Framework
SELECT OPTIONS
Option	Value
Model Class	Employee
DbContext	ApplicationDbContext

Click:

Add
MAGIC HAPPENS HERE

Visual Studio automatically generates:

CRUD operations
Views
Controller code

This is called:

Scaffolding

Run Project

Press:

F5

Open:

/Employees

Create employee
Edit

View list

If Scaffloding fails:

Install the required packages:
Install-Package Microsoft.EntityFrameworkCore.Tools -Version 8.0.8
Install-Package Microsoft.EntityFrameworkCore.Design -Version 8.0.8
Install-Package Microsoft.VisualStudio.Web.CodeGeneration.Design -Version 8.0.7
Install-Package Npgsql.EntityFrameworkCore.PostgreSQL -Version 8.0.4

Install Scaffolding Tool

Run in terminal:

dotnet tool install -g dotnet-aspnet-codegenerator

To see this error : run the below command 
dotnet aspnet-codegenerator controller -name EmployeesController -m Employee -dc ApplicationDbContext --relativeFolderPath Controllers --useDefaultLayout --referenceScriptLibraries   


And the main thing is .csproj file should be inside the folder 
like: folder/filename.csproj 
Not like: folder1/folder2/filename.cs.proj
